서버 제출 테스트 1

    	이름과 나이를 입력받아서 서버 (Java코드)로 제출
    	
    	* 준비물 *
    	
    	1) form 태그 : 내부에 작성된 input 태그 값을 지정된 경로로 제출하는 역할
    		- action : 제출할 경로를 작성하는 부분
    		- method : 제출 방법을 명시 ( GET / POST ) 기본값 : GET
    		제출 == Client 가 서버로 요청하는 것
    		
    	2) input 태그 : 값을 입력하는 양식 제공
    		- name (* 중요 * ) : 서버로 값 제출 시 key 역할을 함
    		- 입력된 값 또는 value : 서버로 제출 시 value 역할 
    		
    		
       	==> /?inputName =홍길동
    
    	** GET : get을 통한 요청은 URL 주소 끝에 파라미터로 포함되어 전송되며,
    	이 부분은 Query String 이라고 부른다
    	방식은 URL 끝에 "?"를 붙이고 그 다음 변수명1 = 값1 & 변수명2 = 값2...
    	ex)www.example.com/?name1=value1&name2=value2...
    	-> 서버에서는 name1 과 name2 라는 파라미터 명으로 각각 value1 과 value2 파라미터 값을 전달받을 수 있다.

        [장점]
         -단순하다
         -캐싱 가능(Chching, 저장)
            -> 주소 저장 == 북마크, 즐겨찾기, 주소 공유 등 가능..
         
        [단점]
         - 글자 수 제한이 있다.
         - 보안에 취약하다(비밀번호등 주소에 보이게 됨)

        GET 방식과는 다르게
        전달되는 값을 주소가 아닌
